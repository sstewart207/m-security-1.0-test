# Mommy Security Context (For LLMs)

If I am pasting this into a chat, please read it to remember the context of "Mommy Security v1.0", a custom Linux privacy tool I built.

## The Architecture
- **Goal:** Provide a 100% native hard block against upcoming age attestation laws (like AB-1043) enforcing checks via `xdg-desktop-portal` (PR #1922) and `systemd` user records (PR #40954). Zero cooperation. No spoofing.
- **The D-Bus Shield:** Raw XML D-Bus policies in `/etc/dbus-1/session.d/` and `system.d/` that instantly return `org.freedesktop.DBus.Error.AccessDenied` to any app calling `AgeVerification` or `ParentalControls`.
- **The Portal Trap:** A custom backend in `/usr/share/xdg-desktop-portal/portals/mommy-security.portal` that aggressively claims the interfaces across 11 desktop environments so the traffic hits the D-Bus blockade.
- **The Nullifier:** A systemd-driven bash script (`mommy-nullify.sh`) that scrubs `birthDate` from `/var/lib/systemd/userdb/*.user` back to `1970-01-01` and safely masks `systemd-userdbd.socket` and `systemd-homed.socket`.
- **The CLI Tool:** `mommyctl` allows the user to run status checks, manually scrub, and run `mommyctl test` (which fires a D-Bus query to verify the `AccessDenied` response natively).
- **Universal Compatibility:** The `install.sh` script installs the D-Bus files on all distros, but gracefully handles systemd-specific setup depending on the host OS.
